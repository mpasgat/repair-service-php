# DECISIONS

1. Использован Laravel без тяжелых пакетов авторизации, чтобы быстро реализовать требования тестового с понятной архитектурой MVC.
2. Для ролей выбран простой столбец `users.role` со значениями `dispatcher`/`master` и middleware `role` для контроля доступа.
3. Для хранения данных выбрана SQLite, чтобы запускаться без отдельного сервера БД и упростить локальный старт через Docker.
4. Для сущности заявок выделена отдельная таблица `repair_requests` с индексом по `status` и `assigned_to` для ускорения списков и фильтрации.
5. Защита от гонки в действии `take` сделана через атомарный `UPDATE ... WHERE status = 'assigned'`; это гарантирует один успешный захват и `409` для повторного запроса.
6. Для предсказуемой проверки добавлены сиды с 1 диспетчером, 2 мастерами и несколькими заявками в разных статусах.
7. Добавлены feature-тесты на критичный функционал: создание заявки и конфликт при повторном "взять в работу".
